== DSP judgment calls

* **Sample type**: f32 throughout (matching V).  f64 is used only for
  intermediate sums in `rmsLevel` and `Normalizer.process` to preserve
  numerical accuracy — identical to the V source.
* **Buffer reuse**: `volumod_process` borrows the caller's sample pointer
  directly via a stack-constructed `AudioBuffer` view — zero allocation on
  the hot path.
* **Hot-path allocations**: none.  All filter state is inline in structs.
  `BebopBridge.processAudio` allocates one temporary `AudioBuffer` clone
  per call via the explicit allocator argument, same semantics as V's
  `core.from_samples`.
* **Envelope follower in dB space**: the compressor processes the dB-domain
  envelope (matching V exactly).  The envelope starts at 0.0 dB, which means
  the first few hundred frames of quiet audio see brief over-compression while
  the release tail converges — this is V's behaviour, not a Zig regression.
